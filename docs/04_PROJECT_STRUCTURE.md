## 1. Philosophy of Structure

### 1.1 Separation of Concerns Deepens Maintainability

> **The most modular codebase has the most visible boundaries.**

**Structure Goals:**

| Goal | How It Manifests | Why It Matters |
|------|-----------------|----------------|
| **Single Responsibility** | Each directory contains one conceptual domain | One change rarely affects the whole project |
| **Interface Isolation** | Clear public vs. internal boundaries | Testability, replacement, understanding |
| **Extensibility** | Room for plugins, characters, themes, packs | Open ecosystem |
| **Composability** | Small, focused modules | Reuse across contexts |

### 1.3 Directory Scope Rules

| Directory | Purpose | Contents | Exclusions |
|-----------|---------|----------|------------|\n| **/core** | Orchestrator, planner, event bus | Orchestrator code, dependencies | Character runtime, UI, file system tools |
| **/ai** | AI agents, model routing | Agent definitions, routing logic | Core orchestrator, UI, memory |
| **/character** | All character-related assets | VRMs, animations, personalities, themes | Memory, skills, plugins |
| **/memory** | All memory implementations | Obsidian integration, episodic, semantic | External services, network |
| **/models** | Model adapters | Local providers, cloud adapters | Core routing, teamplaining |
| **/plugins** | Extension system | Plugin loaders, providers | Character packs, themes |
| **/skills** | Desktop automation | File system, terminal, browser, git | Core orchestrator, high-level planning |
| **/apps** | Desktop UI applications | Overlay, settings, character preview | Core logic, business rules |
| **/assets** | Project assets | Icons, VRMs, sounds, static data | Generated files, compiled assets |
| **/tests** | Test suite | Unit, integration, e2e tests | Production source code |

**Rule:** Never place unrelated concepts in the same directory. If a new concept emerges, **create a new directory**, not a subdirectory within an existing unrelated domain.

File Naming and Organization\n\n### 3.1 Source Files\n\n**Rule:** Source files should be organized by capability, not by type.\n\n```text\n# Good: Organized by capability\nplugins/skill-filesystem/src/filesystem.ts\nplugins/skill-filesystem/tests/filesystem.test.ts\n\nplugins/character-nila/src/character.ts\nplugins/character-nila/tests/character.test.ts\n\n# Bad: Organized by technical stack\nplugins/javascript/utils/filesystem-common.ts\nplugins/javascript/components/filesystem-operations.ts\n```\n\n### 3.2 Test Files\n\n```text\n# Unit Tests\nplugins/skill-filesystem/src/filesystem.ts              # Implementation\nplugins/skill-filesystem/tests/filesystem.test.ts         # Tests\n\n# Integration Tests  \nplugins/skill-filesystem/tests/integration/obsidian.e2e.ts  # End-to-end\n\n# e2e Tests\napps/desktop/tests/e2e/character-interaction.spec.ts\n```\n\n### 3.3 Configuration Files\n\n| File | Location | Purpose |\n|------|----------|---------|\n| `package.json` | Root | Dependencies, scripts, monorepo metadata |\n| `pyproject.toml` | Root (if Python backends) | Python dependencies |\n| `Cargo.toml` | Core orchestrator (Rust side) | Rust dependencies |\n| `.charos/` | Root (project-local config) | Overrides to defaults |\n| `core/orchestrator/config.json` | Core (service-specific) | Orchestrator configuration |\n| `plugins/*/manifest.json` | Each plugin | Plugin metadata |\n\n### 3.4 Generated Files\n\n**Never put generated code in source directories.** Use build directories:\n\n```text\nplugins/*/dist/              # Compiled JavaScript/TypeScript\nplugins/*/target/            # Compiled Rust\nplugins/*/build/             # Build artifacts\nplugins/*/tmp/               # Build temporary files\n```\n\n**Move generated files to:**\n- Root: `dist/`, `build/`, `tmp/`, `.cache/`\n- Per-plugin: `dist/`, `target/`, `obj/`\n\n### 3.5 Temporary and Cache Files\n\n```text\ntmp/                        # Build temporary files\ntmp/{project}/              # Per-project build temp\n.direnv/                    # Development environment\n.env.local                  # Local environment overrides\nnode_modules/               # Package manager cache\ntarget/                      # Rust compilation artifacts\n.ollama/                     # Ollama cache\nsymlinks/                    # Temporary symbolic links\n```\n\n**Rule:** Never commit temporary files. Everything in `.gitignore` stays out of source.\n\n---\n\n## 4.
